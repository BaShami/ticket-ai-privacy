01

Overview

Ticket AI Assistant is an internal Chrome extension built for customer care agents at Make.com. It operates exclusively within the Freshdesk support platform to help agents summarise tickets, draft replies, assess CSAT risk, and find related historical tickets — all without leaving the active ticket view.

This extension is an internal operational tool. It is not distributed to end customers and does not collect any data from Make.com's customers or end users.

This policy applies to all agents who install and use the extension. By using it, you acknowledge the practices described here.

02

Data Collected

The extension accesses the following data categories during normal operation:

Ticket content
Subject line, description, and conversation thread of the currently open Freshdesk ticket. Read from the page DOM and/or Freshdesk's internal API using the agent's live browser session.
Agent identity
Agent email address, stored locally after first use and included in webhook payloads for analytics attribution. Never used for authentication or shared externally.
OpenAI API key
Entered once by each agent and stored in chrome.storage.local on their own device. Used only to call the OpenAI API for Related Tickets keyword generation. Never transmitted to any Make.com-operated server.
Related ticket IDs
Freshdesk ticket IDs returned by internal search queries. Sent to the Make.com backend to fetch full ticket details for relevance ranking.
Usage analytics
Action type, timestamp, agent email, tone selection, and ticket ID sent as anonymous usage events to a separate Make.com analytics scenario. Used solely to understand feature adoption.
Tone preference
Selected reply tone (e.g. formal, friendly) stored in chrome.storage.local as a UI preference. Never transmitted independently.
03

How We Use It

All data access is strictly purpose-limited to the following uses:

AI actions
Ticket content is sent to the Make.com webhook, which passes it to an OpenAI GPT model to generate summaries, reply drafts, next-step suggestions, and CSAT risk assessments. transient
Related tickets
Ticket subject and description are sent to OpenAI (via the agent's own API key) to generate keyword variants. These keywords drive internal Freshdesk searches. Resulting IDs are sent to Make.com for ranking. transient
Analytics
Usage events (action type, agent email, timestamp) are logged to an internal Make.com analytics scenario for feature adoption tracking. No ticket content is included. retained

Data is never used for model training, advertising, profiling, or shared with any party outside the tools listed under Third Parties below.

04

Storage & Retention

Local (device only)
OpenAI API key — stored in chrome.storage.local, never leaves the agent's browser.
Agent email — stored in chrome.storage.local for UI persistence.
Tone preference — stored in chrome.storage.local.
no server copy clears on extension uninstall
In-transit (Make.com)
Ticket content sent to Make.com webhooks is processed in memory and not persisted after the webhook response is returned. Make.com execution logs may retain metadata (not content) per Make.com's own data retention policy. transient processing
Analytics store
Usage events (no ticket content) are stored in the Make.com analytics scenario datastore. Retained indefinitely for internal reporting unless manually purged by an administrator. retained
OpenAI (third party)
Ticket content passed to OpenAI via the Make.com connection is subject to OpenAI's data retention and privacy policy. Enterprise API usage may be configured for zero data retention. The agent's personal API key calls (for keyword generation) are also governed by OpenAI's terms. see OpenAI policy
05

Third Parties

The extension interacts with the following external services:

OpenAI
Receives ticket content (via Make.com) for AI action generation, and receives ticket subject/description (via the agent's own API key) for keyword generation in the Related Tickets feature. Governed by OpenAI's Privacy Policy.
Make.com
Hosts the backend automation scenario. Processes webhook payloads containing ticket content. Subject to Make.com's Privacy Notice. This is an internal Make.com-operated tool.
Freshdesk
The extension reads data from the Freshdesk DOM and calls Freshdesk's internal search API using the agent's active browser session. No additional authentication is introduced; the agent's existing Freshdesk session credentials are used.

No other third-party services receive any data from this extension.

06

What We Don't Collect

The following data is explicitly never collected or transmitted:

browsing history pages outside Freshdesk keystrokes or clipboard customer PII beyond ticket content passwords or auth tokens device identifiers location data data from non-ticket FD pages

The extension only activates on Freshdesk ticket pages (URLs matching */helpdesk/tickets/*). It has no access to any other browser activity.

07

Your Rights

As an internal tool used by Make.com employees, data handling falls within Make.com's employee data policies. Agents have the following options:

Clear local data
All locally stored data (API key, email, tone preference) can be cleared by removing the extension from Chrome, or via Chrome's extension storage management.
Analytics opt-out
Analytics events are tied to agent email. Request removal of your data from the analytics datastore by contacting the CX tooling team.
OpenAI key
Each agent controls their own OpenAI API key. You may revoke or rotate it at any time via the OpenAI dashboard. The extension will prompt for a new key on next use.
